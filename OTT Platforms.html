StreamSphere Analytics: OTT Content & User Engagement Analysis¶

Business Problem¶

OTT platforms such as Netflix, Amazon Prime, and Disney+ host thousands of movies and TV shows across multiple genres, countries, and content categories. With a growing content library, it becomes challenging to understand what type of content attracts users, which genres are most popular, how viewing preferences vary across demographics, and what factors influence user engagement.

Without data-driven insights, OTT platforms may struggle to:

  • Recommend relevant content to users.
  • Invest in high-performing genres and content categories.
  • Understand user viewing behavior.
  • Improve subscriber retention and engagement.

Project Objective¶

The objective of this project is to build a relational database and analytics solution that helps an OTT platform analyze its content library and user engagement patterns.

The project will:

  • Organize OTT content data into a normalized SQL database.
  • Track user viewing activities and ratings.
  • Analyze content performance across genres, countries, and content types.
  • Identify viewing trends and user preferences.
  • Generate business insights to support content strategy and decision-making.

Scope of the Project¶

  • Content Analysis
  • Movies vs TV Shows distribution
  • Genre popularity
  • Country-wise content production
  • Content release trends over time
  • Content ratings analysis

User Engagement Analysis¶

  • Most watched content
  • User watch-time trends
  • Popular genres among users
  • User rating patterns
  • Subscriber activity analysis

Business Performance Analysis¶

  • Subscription plan distribution
  • Revenue estimation by subscription plan
  • User retention indicators
  • Content consumption trends

Data Normalization:¶

  • EDA revealed that the listed_in, cast, director, and country columns contain multiple values within a single record. To eliminate redundancy and achieve database normalization, these attributes were separated into individual dimension tables (Genres, Actors, Directors, Countries) and linked to the Content table through bridge tables (Content_Genres, Content_Actors, Content_Directors, and Content_Countries). This design supports many-to-many relationships and improves data integrity.

Conclusion of Data Cleaning, EDA, and Normalization¶

The Netflix OTT dataset was successfully cleaned and prepared for database design. Missing values, duplicate records, and data quality issues were identified and handled appropriately. Exploratory Data Analysis (EDA) was performed to understand the dataset structure, content distribution, genre patterns, country information, director details, and cast information.

The analysis revealed that several columns such as listed_in, cast, director, and country contained multiple values within a single field, leading to data redundancy and making analysis difficult. To address this issue, the dataset was normalized by separating these multi-valued attributes into individual dimension tables and creating bridge tables to maintain many-to-many relationships.

The normalization process resulted in the following relational structure:

  • Content (8706 records)
  • Genres (42 records)
  • Content_Genres (19117 records)
  • Directors (4988 records)
  • Content_Directors (6968 records)
  • Actors (36141 records)
  • Content_Actors (63439 records)
  • Countries (123 records)
  • Content_Countries (9909 records)

This normalized schema reduces redundancy, improves data integrity, and enables efficient querying and analysis. With the data preparation phase completed, the next step is to design and implement the relational database in MySQL by creating databases, tables, primary keys, and foreign key relationships.

The SQL analysis provided valuable insights into the OTT platform's content library by examining content types, genres, countries, directors, actors, ratings, and release trends. Through the use of aggregate functions, joins, subqueries, Common Table Expressions (CTEs), and window functions, meaningful business questions were answered and key patterns were identified.

The results indicate that content availability is concentrated in specific genres and countries, while a small group of directors and actors contribute significantly to the platform's catalog. Additionally, the steady increase in content over the years reflects the continuous expansion of OTT platforms to meet growing audience demand.

Overall, SQL proved to be an effective tool for transforming raw relational data into actionable business insights, supporting data-driven decision-making and strategic content planning for OTT platforms.

Conclusion:¶

The OTT Analytics project successfully transformed a raw Netflix dataset into a structured and analysis-ready relational database. The project began with data cleaning and preprocessing, where missing values, inconsistencies, and formatting issues were addressed. Exploratory Data Analysis (EDA) was then performed to understand content distribution, release trends, ratings, genres, countries, directors, and actors.

To eliminate data redundancy and improve data integrity, the dataset was normalized into multiple related tables including Content, Genres, Directors, Actors, Countries, and their corresponding bridge tables. This normalization process enabled efficient storage and supported complex many-to-many relationships within the database.

The normalized datasets were imported into MySQL, where a relational database was designed using primary and foreign key constraints. SQL queries ranging from basic aggregations to advanced joins, Common Table Expressions (CTEs), and window functions were executed to generate meaningful business insights. The analysis revealed trends in content growth, genre popularity, country-wise content production, director contributions, actor participation, and platform content distribution.

Visualizations were created to effectively communicate key findings such as Movies vs TV Shows distribution, top-performing genres, leading content-producing countries, yearly content growth, and cumulative expansion of the content library.

Overall, this project demonstrates the complete data analytics workflow, including data cleaning, exploratory analysis, database design, normalization, SQL-based analysis, and data visualization. The insights generated can help OTT platforms better understand content trends, audience preferences, and strategic content planning decisions.
